Project Overview

This project compares six algorithmic strategies for solving Wordle — spanning simple heuristics, reinforcement learning, and deep RL — against the known optimal solution (Bertsimas & Paskov, 2024: 3.421 avg guesses, 100% win rate).

The central finding: Wordle's effective state space is just 431 unique game states across all 2,983 possible target words. This makes it trivially solvable by dynamic programming but paradoxically intractable for deep RL's exploration-based learning. A 543K-parameter DQN was the worst performer; a 431-entry lookup table was the best.

Why DQN Fails at Wordle

Pure DQN cannot learn Wordle effectively. As epsilon decays, the model's poor guesses generate bad replay data, which further degrades training — a distribution shift death spiral. The best honest evaluation achieved just 64.4% win rate, compared to 98.5% for randomly picking from remaining valid words.

DQN v2 attempts to fix this by using the rollout solver as a live teacher during training: exploration follows the rollout's optimal move, with a +20 reward bonus for matching. Epsilon decays from 0.9 to 0.4, maintaining at least 40% teacher guidance throughout training.

Architecture Details

DQN: 417-dim state → 512 → 512 → 130-dim output. Output dot-producted with one-hot word encodings (2983 × 130) to score each word. No remaining-word mask — the model picks from all 2,983 words.

Rollout solver: Memoized POMDP rollout. Only 431 unique game states exist across all target words. Cache built in ~15 minutes. Key = tuple of (guess, feedback) pairs.

Frequency heuristic: Scores candidates by letter frequency × 3× positional frequency. Simple but effective at 99.8% win rate.
